# PlantQuality v1.2 Ownership & SLA Plan

Date: 2026-04-24  
Release target: v1.2.0  
Scope: Ownership, due dates, SLA visibility, overdue logic, action-required state, and dashboard/list ergonomics.

## Purpose

PlantQuality v1.2 should make every defect and 8D report clearly owned, time-bound, and operationally trackable. The product should move from an 8D form workflow into a supplier quality task control loop where OEM and supplier teams can immediately understand:

- who owns the defect,
- who must act next,
- when action is due,
- whether the item is overdue,
- and where attention is needed across the portfolio.

This release builds directly on v1.1, which added stronger 8D validation, audit events, revision lifecycle, and review comment states.

## Implementation Status

Initial implementation has started:

- Ownership and SLA schema fields were added to `Defect`.
- `ActionOwner` enum and additional `DefectEventType` audit events were added.
- Shared SLA helpers were added for active due date and overdue calculation.
- Defect creation, supplier 8D save/submit, OEM approval, and OEM revision request now update action owner and due dates.
- OEM and supplier defect lists now show assignee/action/due/overdue information and basic filters.
- Defect detail now includes an editable Ownership & SLA panel for permitted roles.
- OEM and supplier dashboards now include SLA/action workload cards.

## Current State

PlantQuality currently supports these defect statuses:

- `OPEN`
- `IN_PROGRESS`
- `WAITING_APPROVAL`
- `RESOLVED`
- `REJECTED`

The v1.1 workflow includes:

- server-side 8D completeness validation before supplier submission,
- audit events for defect and 8D workflow activity,
- revision metadata on 8D reports,
- review comment lifecycle with open/resolved states and supplier response,
- OEM review display split for D5, D6, D7 document updates, and D7 preventive actions.

Current operational gaps:

- no global OEM defect owner,
- no supplier assignee,
- no response, submission, review, or revision due dates,
- no computed overdue state,
- no action-required indicator,
- no list filters for SLA or assignment,
- no dashboard metrics for overdue or action-owner workload.

## v1.2 Scope

v1.2 includes:

- OEM defect owner.
- Supplier assignee.
- Response, 8D submission, OEM review, and revision due dates.
- Computed overdue state.
- `Action Required By` indicator with values `OEM`, `SUPPLIER`, and `NONE`.
- Dashboard cards for overdue and action-owner workload.
- OEM and supplier list columns and filters for due dates, overdue state, assignment, and action owner.
- Defect detail SLA panel with owner, assignee, active due date, and overdue state.
- Audit events and notifications when ownership or due dates change.

## Out of Scope

v1.2 does not include:

- per-step evidence attachments,
- PDF or Excel export,
- supplier scorecards,
- AI provenance,
- recurring background escalations or scheduled reminder jobs.

Evidence attachments should follow in v1.2.1 after ownership and due dates are stable.

## Product Rules

### Defect Creation

When an OEM creates a defect:

- `oemOwnerId` defaults to the current OEM user.
- `supplierAssigneeId` is optional at creation.
- `supplierResponseDueAt` defaults to 7 calendar days from creation.
- `currentActionOwner` defaults to `SUPPLIER`.

### Supplier Starts or Saves 8D

When the supplier starts or saves an 8D report:

- `currentActionOwner` remains `SUPPLIER`.
- `eightDSubmissionDueAt` may be tracked separately from `supplierResponseDueAt`.
- If no `eightDSubmissionDueAt` exists, it should default to the existing `supplierResponseDueAt` or 7 calendar days from defect creation.

### Supplier Submits 8D

When the supplier submits an 8D report:

- `currentActionOwner` becomes `OEM`.
- `oemReviewDueAt` defaults to 3 calendar days from submission.
- existing supplier-side due dates remain stored for audit context but are no longer the active due date.

### OEM Requests Revision

When OEM requests revision:

- `currentActionOwner` becomes `SUPPLIER`.
- `revisionDueAt` defaults to 5 calendar days from rejection.
- revision due date becomes the active due date.

### OEM Approves

When OEM approves the 8D report:

- `currentActionOwner` becomes `NONE`.
- overdue state no longer applies.
- active due date should display as closed or not applicable.

### Overdue Logic

A defect is overdue when:

- status is not `RESOLVED`,
- `currentActionOwner` is not `NONE`,
- and the active due date is before today.

Overdue is computed behavior, not a persisted status.

## Implementation Interface Changes

### Schema

Add enum:

```prisma
enum ActionOwner {
  OEM
  SUPPLIER
  NONE
}
```

Extend `Defect` with:

```prisma
oemOwnerId             String?
oemOwner               User?       @relation("DefectOemOwner", fields: [oemOwnerId], references: [id])
supplierAssigneeId     String?
supplierAssignee       User?       @relation("DefectSupplierAssignee", fields: [supplierAssigneeId], references: [id])
supplierResponseDueAt  DateTime?
eightDSubmissionDueAt  DateTime?
oemReviewDueAt         DateTime?
revisionDueAt          DateTime?
currentActionOwner     ActionOwner @default(SUPPLIER)
```

Add reverse relations to `User`:

```prisma
ownedDefects       Defect[] @relation("DefectOemOwner")
assignedDefects    Defect[] @relation("DefectSupplierAssignee")
```

Extend `DefectEventType` with:

```prisma
OWNER_CHANGED
SUPPLIER_ASSIGNEE_CHANGED
DUE_DATE_CHANGED
```

`SLA_OVERDUE_VIEWED` is not required for v1.2.

### Server Helpers

Add shared helpers for:

- `getActiveDueDate(defect)`
- `isDefectOverdue(defect, now)`
- `getActionOwnerLabel(defect)`
- `addCalendarDays(date, days)`
- `startOfToday(now)`

Helper rules:

- `WAITING_APPROVAL` uses `oemReviewDueAt`.
- `REJECTED` uses `revisionDueAt`.
- `OPEN` uses `supplierResponseDueAt`.
- `IN_PROGRESS` uses `eightDSubmissionDueAt` first, then `supplierResponseDueAt`.
- `RESOLVED` has no active due date.

## Server Action Changes

### Create Defect

Update OEM create defect flow:

- set `oemOwnerId` to current user,
- optionally accept `supplierAssigneeId`,
- set `supplierResponseDueAt` default to 7 calendar days,
- set `currentActionOwner` to `SUPPLIER`,
- create audit event with owner, assignee, and due date metadata.

### Save 8D Step

When supplier saves first draft:

- keep `currentActionOwner` as `SUPPLIER`,
- set `eightDSubmissionDueAt` if missing,
- audit due date initialization if changed.

### Submit 8D

When supplier submits:

- set `currentActionOwner` to `OEM`,
- set `oemReviewDueAt` to 3 calendar days from submission,
- log due date/action owner metadata in `DefectEvent`.

### Reject 8D

When OEM requests revision:

- set `currentActionOwner` to `SUPPLIER`,
- set `revisionDueAt` to 5 calendar days from rejection,
- log due date/action owner metadata in `DefectEvent`.

### Approve 8D

When OEM approves:

- set `currentActionOwner` to `NONE`,
- preserve historical due dates,
- log closure metadata.

### Ownership and SLA Updates

Add server action for ownership/SLA updates:

- OEM `ADMIN` and `QUALITY_ENGINEER` can update OEM owner, supplier assignee, and all SLA dates for defects in their company.
- Supplier `ADMIN` can update supplier assignee for defects assigned to their company.
- Supplier `QUALITY_ENGINEER` can assign an unassigned defect to self.
- `VIEWER` cannot mutate ownership or SLA fields.

All mutations must be tenant-scoped by `session.user.companyId`.

## Notifications and Audit

Create notifications when:

- supplier assignee changes,
- OEM owner changes,
- active due date changes,
- action owner changes through submit, reject, or approve transitions.

Create audit events for:

- owner change,
- supplier assignee change,
- due date change,
- action owner transition.

Notification links should point to the relevant defect detail page for the recipient's company type.

## Permissions

Permission rules:

- OEM `ADMIN` and `QUALITY_ENGINEER`: update OEM owner, supplier assignee, and all due dates.
- Supplier `ADMIN`: update supplier assignee for own-company defects.
- Supplier `QUALITY_ENGINEER`: self-assign only when `supplierAssigneeId` is empty.
- `VIEWER`: read-only.

All reads and writes must remain company-scoped.

## Test Plan

### Schema and Generation

- Migration applies cleanly.
- Prisma client generates.

### Workflow

- Creating a defect sets default OEM owner, supplier response due date, and action owner.
- Supplier first save initializes 8D submission due date if missing.
- Supplier submit changes action owner to OEM and sets OEM review due date.
- OEM reject changes action owner to supplier and sets revision due date.
- OEM approve clears operational action state by setting action owner to `NONE`.

### Overdue and Filtering

- Overdue badge appears when active due date is before today.
- Resolved defects are never overdue.
- OEM filters return overdue, action-by-supplier, action-by-OEM, and assigned-to-me records correctly.
- Supplier filters return overdue, assigned-to-me, and waiting-customer-review records correctly.

### Permissions

- OEM permitted roles can update ownership and due dates.
- Supplier admin can update supplier assignee.
- Supplier quality engineer can self-assign only when unassigned.
- Viewer cannot mutate SLA or ownership data.
- Cross-tenant ownership changes are rejected.

### Regression

- Existing 8D validation, submit, reject, approve, review comment, and audit flows still work.
- `npm run lint` passes.
- `npm run typecheck` passes.
- `npm run build` passes.

## Acceptance Criteria

v1.2 is complete when:

- every active defect has a visible action owner,
- every active defect can show an active due date,
- overdue state is visible in lists, detail pages, and dashboards,
- owner and assignee are visible in lists and detail pages,
- permitted users can update ownership/SLA fields,
- all ownership and SLA changes are audit logged,
- assignment and due-date changes notify affected users,
- all validation commands pass.

## Assumptions

- Use calendar days, not business days.
- Store due dates as `DateTime`.
- Display due dates as date-only in the UI.
- Keep `DefectStatus` unchanged.
- Do not persist overdue as a status.
- Evidence attachments move to v1.2.1.
