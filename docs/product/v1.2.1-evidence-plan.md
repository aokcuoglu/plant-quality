# PlantQuality v1.2.1 Evidence & Verification Plan

Date: 2026-04-24  
Release target: v1.2.1  
Scope: 8D evidence attachments, verification readiness, evidence-aware review controls, and audit visibility.

## Purpose

PlantQuality v1.2 introduced ownership, SLA, and action responsibility.  
v1.2.1 should complete the operational loop by adding structured evidence handling so OEM decisions are not made on text-only 8D submissions.

Product framing for this release:

- from "who should act and by when"
- to "who should act, by when, and with which verifiable evidence."

## Current State

Current workflow strengths:

- statuses and action ownership are operational (`OPEN`, `IN_PROGRESS`, `WAITING_APPROVAL`, `RESOLVED`, `REJECTED`),
- due dates and overdue logic are visible,
- review comments and revision loop are active,
- audit events capture major transitions.

Current gap:

- no per-step evidence upload,
- no evidence metadata model,
- no mandatory evidence rule before supplier submission,
- no OEM-side evidence preview for review decisions,
- no evidence lifecycle in audit history.

## v1.2.1 Scope

v1.2.1 includes:

- evidence attachments linked to D3, D5, D6, D7 sections,
- multiple files per section with metadata (name, size, mime, uploader, timestamp),
- tenant-scoped upload and download access control,
- supplier-side evidence management (add/remove before submission, add on revision),
- OEM evidence preview/download in defect detail,
- submission validation rules for required evidence sections,
- audit events for evidence add/remove actions,
- list/detail level evidence completeness indicator.

## Out of Scope

v1.2.1 does not include:

- OCR/AI evidence analysis,
- virus scanning pipeline (manual trust model remains in-app),
- CAD/native file rendering,
- external DMS integration (SharePoint, SAP DMS, etc.),
- scheduled reminder jobs tied to missing evidence.

## Product Rules

### Evidence Section Mapping

Evidence can be attached to these sections:

- `D3` containment evidence,
- `D5` permanent corrective action evidence,
- `D6` validation evidence,
- `D7` preventive action evidence.

### Minimum Submission Requirements

Supplier cannot submit 8D unless:

- at least 1 evidence exists for `D5`,
- at least 1 evidence exists for `D6`,
- and at least 1 evidence exists for `D7`.

`D3` evidence remains optional for v1.2.1.

### File Constraints

Initial constraints:

- max file size: 20 MB per file,
- max 15 files per defect per section,
- accepted mime types: `application/pdf`, `image/png`, `image/jpeg`, `image/webp`.

### Revision Behavior

On OEM revision request:

- existing evidence remains visible,
- supplier can add new evidence and remove supplier-uploaded evidence not referenced by OEM lock rules,
- OEM-uploaded files (if later enabled) cannot be removed by supplier.

### Approval Behavior

OEM approval is blocked if required evidence is missing at decision time.

## Implementation Interface Changes

### Schema

Add enum:

```prisma
enum EightDSection {
  D3
  D5
  D6
  D7
}
```

Add model:

```prisma
model DefectEvidence {
  id            String       @id @default(cuid())
  defectId      String
  section       EightDSection
  storageKey    String
  fileName      String
  mimeType      String
  sizeBytes     Int
  uploadedById  String
  companyId     String
  createdAt     DateTime     @default(now())
  deletedAt     DateTime?

  defect        Defect       @relation(fields: [defectId], references: [id], onDelete: Cascade)
  uploadedBy    User         @relation(fields: [uploadedById], references: [id])

  @@index([defectId, section])
  @@index([companyId, createdAt])
}
```

Extend `DefectEventType`:

```prisma
EVIDENCE_ADDED
EVIDENCE_REMOVED
```

### Storage Contract

Use existing upload provider with deterministic keys:

`defects/{defectId}/evidence/{section}/{uuid}-{sanitizedFileName}`

Persist only metadata and storage key in database.

### Server Helpers

Add helpers for:

- `getEvidenceCountsBySection(defectId)`,
- `hasRequiredSubmissionEvidence(defectId)`,
- `validateEvidenceFile(file)`,
- `canUserAccessEvidence(session, evidence)`.

## API / Server Action Changes

Add server actions:

- `uploadDefectEvidence(defectId, section, file)`,
- `removeDefectEvidence(defectId, evidenceId)`,
- `listDefectEvidence(defectId)`,
- `getDefectEvidenceDownloadUrl(evidenceId)`.

Behavior:

- all actions tenant-scoped by defect company relation,
- soft delete in DB (`deletedAt`) + storage delete best effort,
- evidence add/remove generates audit event payload with `section`, `fileName`, `sizeBytes`.

Integrate into submit flow:

- `submitEightDReport` must call `hasRequiredSubmissionEvidence`,
- return actionable error message listing missing sections.

Integrate into approval flow:

- `approveReport` rechecks required evidence before status transition.

## UI Changes

### Defect Detail (Supplier)

Add `Evidence` panels in D3, D5, D6, D7:

- upload button + drag-drop zone,
- uploaded file list with size/date/uploader,
- remove action for permitted files,
- inline validation errors,
- section count badges.

### Defect Detail (OEM)

Read-focused evidence view:

- sectioned file lists,
- preview for image/pdf when available,
- download action,
- missing required evidence warnings in review panel.

### Submit/Approve Guards

Before supplier submit:

- show missing section checklist modal if requirements fail.

Before OEM approve:

- show blocking message if required evidence is missing.

### List / Dashboard Indicators

Add evidence completeness indicator:

- `READY` when required sections complete,
- `MISSING_EVIDENCE` otherwise.

OEM and supplier lists should support filter:

- `Evidence ready`,
- `Evidence missing`.

## Permissions

- OEM `ADMIN` and `QUALITY_ENGINEER`: view/download all evidence in tenant scope.
- Supplier `ADMIN` and `QUALITY_ENGINEER`: upload/remove evidence for defects in supplier scope.
- Supplier `VIEWER`: read-only evidence access.
- OEM `VIEWER`: read-only evidence access.

Tenant isolation is mandatory for every read/write action.

## Audit & Observability

Audit event payload minimum:

- `section`,
- `fileName`,
- `mimeType`,
- `sizeBytes`,
- `actorUserId`.

Add lightweight metrics:

- evidence upload success/failure count,
- average upload duration,
- missing-evidence submit block count.

## Test Plan

- Prisma migration applies cleanly and client generation passes.
- Upload accepts supported file types and rejects unsupported types.
- File size limit enforcement works.
- Supplier submit is blocked when D5/D6/D7 required evidence is missing.
- OEM approve is blocked when required evidence is missing.
- Evidence list returns only tenant-scoped files.
- Unauthorized remove/download attempts are rejected.
- Audit events are written on add/remove.
- `npm run lint`, `npm run typecheck`, and `npm run build` pass.

## Assumptions

- Existing storage provider remains the same in v1.2.1.
- Soft-delete is sufficient for audit needs in this release.
- Required evidence policy stays section-based, not step-field-based.
- Evidence retention policy is inherited from current tenant policy.

## Release Exit Criteria

v1.2.1 is release-ready when:

- required evidence gating is active in both submit and approve flows,
- evidence files are visible and downloadable by permitted roles,
- audit trails cover add/remove lifecycle,
- and no cross-tenant evidence leakage exists in tests.
