## 2. Plan Ownership Model

- **Plan is company-level.** The `Company.plan` field determines the plan for all users in that company.
- **Users inherit the company plan.** The `User.plan` field is a legacy fallback; the authoritative source is `Company.plan`.
- **Session reads company plan.** `session.user.plan` is populated from `user.company.plan` in the JWT flow.
- **BASIC normalizes to FREE.** The `normalizePlan()` function maps `BASIC` → `FREE` for backward compatibility.
- **App tier values:** `FREE`, `PRO`, `ENTERPRISE` (these are the only three meaningful tiers).
- **Supplier companies are FREE** but supplier users follow `supplierAccess` rules instead of plan tier.

## 6. Known Limitations

- **No real billing:** Upgrade CTAs are placeholders. No Stripe integration.
- **No invoices:** No billing or invoice management exists.
- **No SSO:** SSO is a gate placeholder only. No SAML/OIDC implementation.
- **No API access:** API routes are gate placeholders for Free/Pro.
- **No Webhooks:** Webhooks are gate placeholders.
- **No ERP integration:** No SAP or external system integration.
- **No Multi-Plant:** Multi-plant is a gate placeholder.
- **Usage consumption:** Not all actions increment usage counters yet. Defect and field defect creation counters may not auto-increment.
- **Email notifications:** Enterprise-only gate, not implemented.
