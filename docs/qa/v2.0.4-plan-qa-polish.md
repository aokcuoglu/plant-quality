## 1. Medium Issues Fixed from v2.0.3 Review

### M1: Feature Access matrix showed raw enum keys

**Before:** The Plan & Usage page Feature Access section showed raw `minPlan` enum values (`FREE`, `PRO`, `ENTERPRISE`) next to locked features.

**After:** Uses `PLAN_LABELS[f.minPlan]` to display formatted labels: "Free", "Pro", "Enterprise".

**File:** `src/app/(dashboard)/oem/settings/plan/page.tsx`

### M2: Usage display showed misleading "0/—" with red bar for blocked features

**Before:** FREE plan usage rows for blocked features (limit=0, e.g. AI Classification, War Room) showed `0 / —` with a red progress bar at 100%, implying the user had exceeded a limit they never had.

**After:** When `limit === 0`, the usage row shows only `—` (em dash) with no count and no progress bar, clearly indicating the feature is not available at the current plan level.

**File:** `src/app/(dashboard)/oem/settings/plan/page.tsx`

### M3: Storage label showed raw "Storage (MB)" instead of friendly label

**Before:** `Storage (MB)` label with raw MB counts (e.g. `450 / 200,000`).

**After:** Label simplified to `Storage`. Values remain in MB for precision; Enterprise shows "Unlimited".

**File:** `src/app/(dashboard)/oem/settings/plan/page.tsx`
