## Demo Scenario Map

### Scenario A — Strong Linkage

**Setup:** Same OEM (PlantX Automotive), same supplier (Precision Parts), same part number (AX-7420-B).

**Records sharing this part/supplier:**

| Record | ID | Type | Status | Key field |
|--------|----|------|--------|-----------|
| Field Defect | `fd-linkage-a` | FIELD_DEFECT | SUPPLIER_ASSIGNED | partNumber=AX-7420-B, category=Casting porosity |
| Defect | `defect-001` | DEFECT | OPEN | partNumber=AX-7420-B |
| Defect (resolved) | `defect-004` | DEFECT | RESOLVED | partNumber=AX-7420-B |
| PPAP | `ppap-001` | PPAP | SUBMITTED | partNumber=AX-7420-B |
| IQC (rejected) | `iqc-001` | IQC | COMPLETED / REJECTED | partNumber=AX-7420-B |
| FMEA | `fmea-001` | FMEA | SUPPLIER_IN_PROGRESS | partNumber=AX-7420-B |

**Test pages (OEM Pro):**

- `/quality/oem/field/fd-linkage-a` — Should show related PPAP, IQC (with IQC_REJECTION badge), FMEA (with FMEA_COVERAGE badge), Defect
- `/quality/oem/iqc/iqc-001` — Should show related PPAP, FMEA, Field Defect (fd-linkage-a), Defect
- `/quality/oem/fmea/fmea-001` — Should show related Field Defect (fd-linkage-a), IQC (with IQC_REJECTION), PPAP, Defect
- `/quality/oem/ppap/ppap-001` — Should show related IQC (with IQC_REJECTION), FMEA, Field Defect (fd-linkage-a), Defect
- `/quality/oem/defects/defect-001` — Should show related Field Defect (fd-linkage-a), IQC, PPAP, FMEA

**Expected reason badges:**

| Link | Badge(s) |
|------|----------|
| fd-linkage-a ↔ PPAP ppap-001 | Same part number, Same supplier + part |
| fd-linkage-a ↔ IQC iqc-001 | Same part number, Same supplier + part, IQC rejection history |
| fd-linkage-a ↔ FMEA fmea-001 | Same part number, Same supplier + part, FMEA coverage |
| fd-linkage-a ↔ Defect defect-001 | Same part number, Same supplier + part |

**Confidence:** At least "Moderate" (score >= 60 for same-part + same-supplier combos). IQC rejection and FMEA coverage variants will be "Strong" (score >= 70).

### Scenario C — FMEA Coverage

**Setup:** FMEA fmea-001 has a row with `failureMode: "Mold surface contamination"` and `failureEffect: "Surface porosity defect on casting"`. Field defect fd-linkage-a has `category: "Casting porosity"`.

**Expected:** When viewing fd-linkage-a, the FMEA fmea-001 should appear with the `FMEA_COVERAGE` badge because of keyword overlap between "Casting porosity" (category) and "Cylinder Head Casting Process FMEA" (title) or the category matching the failure effect keywords.

**Test:**

1. `/quality/oem/field/fd-linkage-a` → FMEA section should show fmea-001 with FMEA_COVERAGE badge
2. `/quality/oem/fmea/fmea-001` → Field Defect section should show fd-linkage-a

**Note:** FMEA coverage badge requires `keywordOverlapScore >= TITLE_KEYWORD_STRONG` (3+ overlapping tokens after stop-word filtering and minimum 4-char token length). The specific overlap depends on the tokenization logic. Even without FMEA_COVERAGE, fmea-001 will still appear due to SAME_PART + SAME_SUPPLIER.

### Scenario D — IQC Rejection History

**Setup:** IQC iqc-001 has `result: "REJECTED"` for part AX-7420-B from Precision Parts.

**v2.5.2 Bug (fixed in v2.5.3):** `isIqcRejectedOrOnHold()` was called with `i.status` (IqcStatus) instead of `i.result` (IqcResult). Since `status: "COMPLETED"` never matches "REJECTED", the IQC_REJECTION badge never appeared.

**Fix:** All three occurrences in `find-related.ts` now check `i.result` instead of `i.status`.

**Expected after fix:** IQC iqc-001 should appear with `IQC_REJECTION` badge when viewing related records for any record with the same part and supplier.

**Test:**

1. `/quality/oem/field/fd-linkage-a` → IQC section should show iqc-001 with "IQC rejection history" badge
2. `/quality/oem/ppap/ppap-001` → IQC section should show iqc-001 with "IQC rejection history" badge
3. `/quality/oem/fmea/fmea-001` → IQC section should show iqc-001 with "IQC rejection history" badge

**Negative test:** IQC iqc-002 (result: ACCEPTED, part BR-1122-C) should NOT show IQC_REJECTION badge.

### Scenario E — Weak False-Positive

**Setup:** Field defect fd-weak-e has `partNumber: "WM-3300-A"`, `supplierId: "supplier-company"` (Precision Parts). This is a completely unrelated part from the same supplier.

**Expected:** When viewing other Precision Parts records (e.g., defect-001, iqc-001, ppap-001), fd-weak-e should NOT appear in the related panel because:

- No same-part match (WM-3300-A ≠ AX-7420-B)
- Same-supplier-only score is 15, below the threshold of 50
- No keyword overlap on failure mode

**Test:**

1. `/quality/oem/defects/defect-001` → Field Defects section should NOT include fd-weak-e
2. `/quality/oem/iqc/iqc-001` → Field Defects section should NOT include fd-weak-e
3. `/quality/oem/ppap/ppap-001` → Field Defects section should NOT include fd-weak-e

**Note for Supplier A pages:** Same logic applies — when Supplier A views their details, fd-weak-e also shares same supplier but different part, so it would only score 15 → filtered. This is correct behavior.

## Bug Fixes in v2.5.3

### IQC Rejection Badge Not Showing (Critical)

- **Problem:** `isIqcRejectedOrOnHold()` was called with `i.status` (IqcStatus enum like "COMPLETED") instead of `i.result` (IqcResult enum like "REJECTED"). Since no IqcStatus value matches "REJECTED" or "ON_HOLD", the IQC_REJECTION badge never appeared for any IQC record.
- **Fix:** Changed all 3 occurrences in `find-related.ts` from `i.status` to `i.result`.
- **Lines:** ~311, ~892, ~1089 in `src/lib/quality-linkage/find-related.ts`
